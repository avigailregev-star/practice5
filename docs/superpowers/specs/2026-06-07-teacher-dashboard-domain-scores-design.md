# Teacher Dashboard — Domain Scores + AI Weekly Summary

**Date:** 2026-06-07
**Scope:** practice5 Next.js app — teacher dashboard
**Goal:** Show per-domain assessment scores (notes/rhythm/pitch) on each student card, and let the teacher generate an AI weekly summary per student via a button click.

---

## Overview

The existing teacher dashboard shows a student list with session history. This feature adds:
1. **Domain scores** — average % from the `assessments` table, shown as 3 compact colored boxes per student card
2. **AI weekly summary** — a short Hebrew paragraph generated by Claude on demand, saved to `profiles` and displayed persistently

---

## DB Schema Change

Add 2 nullable columns to `profiles`:

```sql
ALTER TABLE profiles
  ADD COLUMN weekly_summary      TEXT         DEFAULT NULL,
  ADD COLUMN summary_updated_at  TIMESTAMPTZ  DEFAULT NULL;
```

- `NULL` = no summary generated yet
- Updated by the Server Action after each Claude call

## Domain Score Display

### Data source

Query the `assessments` table for each student. Group by `type` and compute `AVG(score)`:

```sql
SELECT student_id, type, AVG(score) AS avg_score
FROM assessments
WHERE student_id = ANY($studentIds)
GROUP BY student_id, type
```

Types: `notes`, `rhythm`, `pitch`. Score is 0–100 integer (percentage).

### Color coding

| Score | Color |
|-------|-------|
| ≥ 80% | Green (`text-green-600`, `bg-green-50`) |
| 50–79% | Amber (`text-amber-500`, `bg-amber-50`) |
| < 50% | Red (`text-red-500`, `bg-red-50`) |
| No data | Gray, shows "—" |

## AI Weekly Summary

### Trigger

Teacher clicks a "רענן" button next to the summary in each student's card. The button shows a spinner while the Server Action runs (~2–5 seconds). After completion, the card updates with the new summary and timestamp.

### Server Action: `app/actions/summary.ts`

```ts
"use server";
export async function refreshWeeklySummary(studentId: string): Promise<string>
```

Steps:
1. Authenticate caller (must be teacher role)
2. Fetch from Supabase:
   - `profiles`: student name
   - `assessments`: all rows for the student (type + score)
   - `practice_sessions`: last 10 completed sessions (completed_at, skill_type, duration_minutes, self_rating)
3. Compute per-domain averages from assessments
4. Build Hebrew prompt (see below)
5. Call `claude-haiku-3-5` via Anthropic SDK
6. Save result to `profiles.weekly_summary` + `profiles.summary_updated_at = now()`
7. Return the summary string

### Claude Prompt

```
אתה עוזר למורה לכינור. כתוב סיכום שבועי קצר (2–3 משפטים בלבד) על תלמיד/ה, בעברית.
התמקד בנקודות חוזק ובתחום אחד לשיפור. אל תכלול מספרים ישירות — הסק מהם מסקנות.

נתוני התלמיד/ה:
- שם: {name}
- ציון ממוצע תווים: {notes_avg}% (מתוך {notes_count} מבחנים)
- ציון ממוצע מקצב: {rhythm_avg}% (מתוך {rhythm_count} מבחנים)
- ציון ממוצע גובה צליל: {pitch_avg}% (מתוך {pitch_count} מבחנים)
- אימונים בשבוע האחרון: {session_count} ({total_minutes} דקות)
- תחומים שתורגלו: {skill_types}

כתוב רק את הסיכום, ללא כותרת וללא פתיחה כמו "סיכום:".
```

### Model

`claude-haiku-3-5` — fast and cost-effective for short Hebrew text generation.

### No summary state

When `weekly_summary IS NULL`, the card shows:
> "לחצי על רענן לייצור סיכום שבועי 🤖"

Timestamp (`summary_updated_at`) is shown as "עודכן: DD.MM.YYYY" beneath the summary.

## Non-Goals

- No auto-scheduling of summaries (teacher triggers manually)
- No summary for the teacher herself (students only)
- No edit/override of the AI summary by the teacher
- No history of past summaries (only latest is stored)
